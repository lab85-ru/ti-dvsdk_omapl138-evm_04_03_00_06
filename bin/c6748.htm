TMS320C6748 Software Components and Installation Guide

This guide will provide you an overview of the available components and installation information to begin your TMS320C6748 development.

Review the instructions in the Zoom OMAP-L138 EVM User Guide for how to connect the TMS320C6748 SOM to the baseboard and setup/install Code Composer Studio (CCS) to communicate to the device via the JTAG emulation.

Components

The following is a list of the development software components and tools available on the TMS320C6748 device. All the components are available for download from the Software and Development Tools section on the TMS320C6748 device page

  Foundational Software Libraries

    1. C64x+ (fixed) and C674x (floating-point) DSP Library (DSPLIB) - A set of optimized functions that serve as building blocks for various signal processing applications.

    2. C64x+ Image Library (IMGLIB) - A collection of building block kernels used for image and video processing applications.

    3. C64x+ IQMATH - A collection of highly optimized and high precision fixed point mathematical functions that support variable Q point format for the variables.

    4. C67x Fast Run-Time Support (fastMath) library - A collection of optimized floating-point math functions for C programmers.

  Application components

    5. BIOS Platform Support Package (BIOSPSP) - A full compliment of DSP/BIOS based device drivers (I2C, McAsp, SPI, UART, etc.)

    6. EDMA3 Low Level Driver (EDMA3 LLD) - A device driver for using and safely sharing EDMA3 resources in an application.

    7. DSP/BIOS USB package (BIOSUSB) - A JUNGO USB stack capable of USB Host or USB Driver functionality leveraging DSP/BIOS.

    8. Network Developer's Kit (NDK) - A DSP/BIOS based TCP/IP network stack and services.

    9. Network Support Package (NSP) - A platform specific network device driver support for NDK development.

    10. DSP/BIOS Real-time File System (RTFS) - A POSIX-compliant embedded FAT file system for devices running DSP/BIOS

  Multimedia components

  For multimedia support (video, image, speech, and audio) on the TMS320C6748 (DSP-only) device the following components are available:

    11.Multimedia Framework Product (MFP - Codec Engine w/ cetools) - A collection of software components which includes Codec Engine to invocate and instantiate XDAIS-compliant algorithms and XDM-compliant codecs.

    Various C64x+ DSP core-based XDM-compliant Video, Image, Speech and Audio Codecs available from TI here.

  Tools

    12. C6000 Code Generation Tools (CGTools) - A foundational tools for building applications for the C6000 DSP core.

    13. C6Flo Graphical Development Tool - A tool to create working C6000 DSP applications from an intuitive block diagram view..

    14. PRU Software Development Package - A required tools for developing software to leverage the Programmable Realtime Unit Subsystem

Additional required components (DSP/BIOS v5.41 and XDCTools v3.20) for DSP build and development are provided as part of the CCS v4.2 installer.

Visit the TMS320C6748 device page for a complete list of DSP software components and tools as more become available for this device.

Installation

A DVD with CCS v4.2 has been provided in the kit. Install CCS v4.2 (default recommended - C:\Program Files\Texas Instruments) before proceeding with the TMS320C6748 components installation.

NOTE: Refer to the CCSv4 Windows 7 support FAQ page for information on possible conflicts when installing and running on Windows 7.

CCS v4.2 includes the CGTools v7.0.x. Most component libraries and examples have been built and validated using CGTools v6.1.x. We recommend downloading and installing the most recent CGTools v6.1.x as this will provided a cleaner rebuild process for the various components and examples provided. All components can be rebuilt using the CGTools v7.0.x if desired. There are no known issues using CGTools v7.0.x at this time.

The various components listed come in different installation packages(exe, tar.gz, zip, zip w/ exe).

It is recommended that all components be installed in the same directory as CCSv4 (C:\Program Files\Texas Instruments). If a non default directory is selected ensure you specify a unique directory during the installation of the component. Otherwise all components will be installed in the same base directory.

1. For installation of components with self installers (*.exe), run the installer, follow the installation instructions and set the Destination Folder to the same directory as your CCSv4 installation. If installing in a non default directory ensure a unique directory is selected for the Destination Folder. For CGTools, you might specify something similar to the following: C:\my_ccsv4_install_dir\C6000 Code Generation Tools 6.1.17)

2. For installation of components in ZIP format (*.zip), install the component by extracting the file into the directory where CCSv4 was installed. As part of the extraction a unique directory will be created (e.g. ndk_2_20_00_19). If the ZIP file contains an installer (*.exe), extract the file and run the installer as per step 1.

3. For installation of components as tarball (*.tar.gz), install the component by extracting the file into the directory where CCSv4 was installed. As part of the extraction a unique directory will be created (e.g. rtfs_1_10_02_32 directory will be created.

For more information on the components, refer to the Release Notes and within the docs directory of the components installation/extracted directory. Additional information can also be found in the Getting Started Guide (GSG) for the TMS320C6748.

Starting your Development on the TMS320C6748 device

Most of the components installed contain a mixture of CCSv3.3 and/or CCSv4 example projects that use DSP/BIOS 5.41.x, CGTools 6.1.x, and XDCTools 3.20.x. Each component installed requires a its own set of environment variables that need to be defined for the application example projects to locate it's dependent components.

Open the Environment Variables window, to do so Right-click "My Computer" and select Properties. From the Advanced tab select Environment Variables. For the components above, here is a list of the environment variables need to be defined. NOTE: Not all components require all the variables, refer to the Release Note or docs folder for a given component for more information on its dependencies.

    BIOS_INSTALL_DIR = C:\Program Files\Texas Instruments\bios_5_41_07_24

    PSP_DRIVERS_INSTALL_DIR = C:\Program Files\Texas Instruments\pspdrivers_01_30_01

    NDK_INSTALL_DIR = C:\Program Files\Texas Instruments\ndk_2_20_00_19

    NSP_INSTALL_DIR = C:\Program Files\Texas Instruments\nsp_1_00_00_09

    EDMA3_BIOS5_INSTALLDIR = C:\Program Files\Texas Instruments\edma3_lld_01_11_01_04

    XDCROOT = C:/Program Files/Texas Instruments/xdctools_3_20_03_63

Some of the variables might already be defined in your system due to previous installed versions of the same components. Make appropriate adjustments to the variable to reflect the installation location of the new components.

  Importing CCSv4 projects

    1. To open a component's CCSv4 projects, start CCS.

     If not previously started after installing the components, you will be prompted to enable the new components discovered extension site and restart CCS.

    2. Select "Project->Import Existing CCS/CCE Eclipse Project" from the menu.

    3. Uncheck the "Copy projects into workspace" option. Enable "Select search-directory" option and click Browse.

    4. Point to the root directory of a component installed (e.g. C:\Program Files\Texas Instruments\pspdrivers_01_30_01) and click OK. CCS will scan the selected directory to find all available CCSv4 projects.

    For the NSP examples enable the "Select archive file" option in step 3. Browse to C:\Program Files\Texas Instruments\nsp_1_00_00_09\packages\ti\ndk\examples directory and select ndk_evm6748_bios5_examples.zip file.

    5. Enlarge the Import project window to see all available projects options to import. Select the correct projects for your given device (C6748 or evm6748) and click Finish. NOTE: Projects with the same name can not be opened up simultaneously in the same workspace.

    6. Once projects have been imported, set the desired project to be built to Active by Right-clicking on the project name and selecting "Set as Active Project". Then select Project->Rebuild Active Project from the menu to begin rebuilding.

    Most CCSv4 project were create to use CGTools v6.1.x. If CGTools v6.1.x was not installed a build error message will appear in the Problems window. You can either install CGTools v6.1.x or changing the settings for each project properties to use CGTools v7.0.x NOTE: Only the included NSP CCSv4 example projects have the properties set to use the CGTools v7.0.x, most other components must be changed manually as previously stated.

  Importing CCSv3.3 projects

    It is recommended to import the CCSv4 projects instead if available for the component.

    1. To open a component's CCSv3.3 projects(*.pjt), start CCS.

     If not previously started after installing the components, you will be prompted to enable the new components discovered extension site and restart CCS.

    2. Select "Project->Import Legacy CCSv3.3 Project" from the menu.

    3. Select "Keep original location for each project" option and Enable "Create a subfolder for each Eclipse project". Enable "Select search-directory" option and click Browse.

    4. Point to the root directory of a component installed and click OK. CCS will scan the selected directory to find all available CCSv4 projects.

    5. Enlarge the Import project window to see all available projects options to import. Select the correct projects for your given device (C6748 or evm6748) and click Next. NOTE: Projects with the same name can not be opened up simultaneously in the same workspace.

    6. Select the Code Generation Tools version for each project and click Next.

    7. Select "Automatically determine DSP/BIOSv5.x support enablement" and click Finish.

    8. Once projects have been imported, set the desired project to be built to Active by Right-clicking on the project name and selecting "Set as Active Project". Then select Project->Rebuild Active Project from the menu to begin rebuilding.
