Welcome

Thank you for choosing the OMAP-L138 eXperimenter or Evaluation Module (EVM) kit.  Please refer back to this page if you are not sure how to proceed with your evaluation.  Its purpose is to easily provide the information you need while evaluating the OMAP-L138. 

IMPORTANT NOTE - The Digital Video Software Development Kit (DVSDK) on the OMAP-L138 is for Linux Development only. This kit is only supported on an Ubuntu 10.04 LTS host machine. A CD with the supported Linux distribution is provided in the kit.  If you don't have a dedicated host Linux PC to install Ubuntu 10.4, visit How to create a Linux host machine on your Windows PC.

Development for the TMS320C6748 device is supported on a host Windows PC only. Installation information can be found in below.

Starting your Development on OMAP-L138

1.  Hopefully you have already enjoyed the demos described in the OMAP-L138 eXperimeter Quick Start Guide or the OMAP-L138 EVM Quick Start Guide. This is the same print out which is provided with your OMAP-L138 kit. To get started ensure you have a Linux Ubuntu 10.04 LTS host environment setup to proceed. A Ubuntu 10.04 CD is provided in the kit. If you don't have a dedicated host Linux PC to install Ubuntu 10.4, visit How to create a Linux host machine on your Windows PC.

2.  Ensure that your board is set up as described in the Quick Start Guide.

3.  Enter the /media/START_HERE directory and execute the dvsdk_omapl138-evm_4_XX_XX_XX_setuplinux file (if necessary, replace 4_XX_XX_XX with the actual version number of the file you have on the SD card). Follow the installer instructions.

host# cd /media/START_HERE

host# ./dvsdk_omapl138-evm_4_XX_XX_XX_setuplinux

4.  Once installation is complete, the Software Developer's Guide (SDG) will open as a pdf file. Follow its instructions to start developing software for the OMAP-L138. The SDG and other documentation can be found in the docs folder of the DVSDK installation directory.

Starting your Development on a TMS320C6748 (DSP-only)
