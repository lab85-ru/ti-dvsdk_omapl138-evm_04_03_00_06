config Settings.defaultMemSegId  // module-wide
index URL

Default ID of the memory segment from which to allocate space (for Memory_alloc/free() calls, as opposed to Memory_segAlloc/Free()). Typically it's the segment where all other code and data go

XDCscript usage meta-domain
Settings.defaultMemSegId = String null;
 
DETAILS
If left as "null", the system will try to find a BIOS segment named "DDR" or "DDR2" and assign its name to this field. In the future this auto-find feature may be removed, so it's best that user .cfg script set this value directly to what they know is appropriate segment.
Only set this parameter if not using ti.sdo.osal.Global.runtimeEnv to set the target OS.

config Settings.traceBufferSize  // module-wide
index URL

Size of the server trace buffer, in MAUs

XDCscript usage meta-domain
Settings.traceBufferSize = Int 32768;
 
DETAILS
The server trace buffer is a circular buffer of characters written to by clients of ti.sdo.ce.osal.Trace (and ti.sdo.utils.trace.GT) and read by the Engine_*Trace() methods.
All the trace data in the target memory is stored into a globally visible character buffer Char Global_traceBuffer[ <traceBufferSize> ];
Only set this parameter if not using ti.sdo.osal.Global.runtimeEnv to set the target OS.
