    33    /*!
    34     *  ======== package.xdc ========
    35     *  Simple XDM IVIDENC1-compatible video encoder.
    36     *
    37     *  This example codec provides a reference for two features of XDM.  Also,
    38     *  It implements the IDMA3 interface, using the Framework Components' ACPY3
    39     *  component to utilize DMA.  (Note the comments in the examples' package.bld,
    40     *  which describe that this dependency on ACPY3 is determined at build-time,
    41     *  and the DMA-based "copy" is only built if ACPY3 is found in the system.
    42     *  More specifically, it is only build if the ti.sdo.fc.acpy3 package is found
    43     *  on the builder's package path.)
    44     *
    45     *  First, it demonstrates a simple implementation of the IVIDENC1 interface.
    46     *  The implementation of the process() function is a simple memcpy.
    47     *
    48     *  The second feature demonstrated is the ability to extend XDM's base
    49     *  interfaces.  In addition to supporting the base interface, it extends
    50     *  IVIDENC1_InArgs with a codec-specific IVIDENC1CPY_InArgs.  This "InArgs"
    51     *  struct defines an additional "maxBytes" field, which enables the
    52     *  application to limit the number of bytes to copy into the output buffer.
    53     */
    54    package ti.xdais.dm.examples.videnc1_copy [2, 0, 0] {
    55        module VIDENC1_COPY;
    56    }
